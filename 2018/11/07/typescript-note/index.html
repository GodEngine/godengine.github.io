typescript 笔记

tsconfig-compilerOptions

target

指定ts编译后的js版本，默认生成的js版本为’es3’，可指定为’es5’、’es6’、’es2017’、’esnext’.

module

js 模块加载机制分别有commonjs(node)、amd(requirejs)、umd等es6原生模块等等，module指定ts编译后的js模块类型，当target值为’es3’和’es5’时，module默认值commonjs，否则为’es6’

moduleResolution typescript包寻址机制

moduleResolution的值可以为’node’或者’Classic’，默认为后者，代表两种不同的包寻址策略，用来告诉编译器在编译import a from "path"时如何寻找path中模块。

首先了解下relative和non-relative，import a from "path"时，path可以是带路径(relative)的或不带路径(non-relative)的。以/root/src/folder/A.ts文件中的import { b } from "./moduleB"和import { b } from "moduleB"为例，寻址机制如下

classic模式下path为relative时的包寻址机制：

1. /root/src/folder/moduleB.ts
2. /root/src/folder/moduleB.d.ts

classic模式下path为non-relative时,机制为从import所在的文件开始按目录向上查找：

1. /root/src/folder/moduleB.ts
2. /root/src/folder/moduleB.d.ts
3. /root/src/moduleB.ts
4. /root/src/moduleB.d.ts
5. /root/moduleB.ts
6. /root/moduleB.d.ts
7. /moduleB.ts
8. /moduleB.d.ts

node模式下typescript模拟node中包文件带路径的寻址机制，即path为relative时的包寻址机制：

1. /root/src/moduleB.ts
2. /root/src/moduleB.tsx
3. /root/src/moduleB.d.ts
4. /root/src/moduleB/package.json (查找"types"定义的路径)
5. /root/src/moduleB/index.ts    
6. /root/src/moduleB/index.tsx
7. /root/src/moduleB/index.d.ts

node模式下typescript模拟node中包文件不带路径的寻址机制，即path为rnon-elative时的包寻址机制：

1. /root/src/node_modules/moduleB.ts
2. /root/src/node_modules/moduleB.tsx
3. /root/src/node_modules/moduleB.d.ts
4. /root/src/node_modules/moduleB/package.json (查找"types"定义的路径)
5. /root/src/node_modules/moduleB/index.ts    
6. /root/src/node_modules/moduleB/index.tsx
7. /root/src/node_modules/moduleB/index.d.ts
1. /root/node_modules//moduleB.ts
2. /root/node_modules//moduleB.tsx
3. /root/node_modules//moduleB.d.ts
4. /root/node_modules//moduleB/package.json (查找"types"定义的路径)
5. /root/node_modules//moduleB/index.ts    
6. /root/node_modules//moduleB/index.tsx
7. /root/node_modules//moduleB/index.d.ts
