typescript 笔记

一些关键字解释

declare: declare module ... 或declare const X意味着在整个编译环境中引入一个全局的声明，declare声明后意味着跟在后面的值为ambient declarations*，即global。比如外部引用了一个js写的库，要在ts中引用这个js库，则可以declare声明这个库的全局变量，使其可以在ts中使用。

type: 在定义复杂类型时，用type来给类型起一个别名，以便重复使用或方便引用。复杂值比如多个值的类型、多种类型、交差类型等。

as： 在将对象字面量{a: 'a'}作为参数或赋值给变量时，如果属性中有类型中不包含的属性，可以用as type绕开检查，绕开属性范围的检测还包括：

interface： 接口用来描述对象、function、类的type，描述function时，参数名称不必一致，是按位置依次判断类型的。接口中，数字类型索引的返回值必须是字符串类型索引返回值的子类型。

this: 当对象内部的函数类型的值中使用this时，如果想将this绑定到当前对象上，则使用箭头函数，但是如果tsconfig使用了
--noImplicitThis配置，则会报错，此时需要在函数的首个参数位置上，用this: type的形式，显式的指定this的指向，避免noImplicitThis错误及获取到this操作返回的真正类型（而非any）

keyof: keyof也称为’ index type query’，可以获取到一个类型的所有key值，即角标值。与泛型配合可以在动态的获取对象的值时拿到正确的类型。

tsconfig-compilerOptions

target

指定ts编译后的js版本，默认生成的js版本为’es3’，可指定为’es5’、’es6’、’es2017’、’esnext’.

module

js 模块加载机制分别有commonjs(node)、amd(requirejs)、umd等es6原生模块等等，module指定ts编译后的js模块类型，当target值为’es3’和’es5’时，module默认值commonjs，否则为’es6’

moduleResolution typescript包寻址机制

moduleResolution的值可以为’node’或者’Classic’，默认为后者，代表两种不同的包寻址策略，用来告诉编译器在编译import a from "path"时如何寻找path中模块。

classic模式下path为relative时的包寻址机制：

1. /root/src/folder/moduleB.ts
2. /root/src/folder/moduleB.d.ts

classic模式下path为non-relative时,机制为从import所在的文件开始按目录向上查找：

当path为relative且path中的module为ambient module declaration(declare module "url")时，import无法解析该module.而当path为non-relative时，通过指定baseUrl选项，import是可以解析ambient module declaration的。在引入外部依赖时，建议使用non-relative的path。

baseUrl

设置non-relative引入的module路径为相对于baseUrl，且当设置path属性时，baseUrl必须存在，当baseUrl不为’.’(及tsconfig所在的目录)时，相应的path也需要作出对应的更改。

rootDirs

如果rootDirs的子目录文件中出现import一个relative module时，编译器会尝试查询整个rootDirs指定的目录，及可以在这些子目录中用’./modulex’查找整个子目录下的modulex。相当于一个虚拟的同级目录。

{
  "compilerOptions": {
    "rootDirs": [
      "src/views",
      "generated/templates/views"
    ]
  }
}

experimentalDecorators

如果代码用到了处于stage2的Decorator装饰器语法，那么需要开启experimentalDecorators选项。

lib

lib为是typescript开启的es功能选项列表。默认情况下，当你的target是es5时，编译器会自动引入DOM、ES5、ScriptHost；当target为es6时，会引入DOM,ES6,DOM.Iterable,ScriptHost。当然如果你target是es5，但是生成的代码运行环境又支持Promise，那么就可以在lib中增加ES2015.Promise的选项。es6/es2015是同义的，只不过es2015由各个功能级别的d.ts文件组成，可以实现更加细颗粒度的功能引用。

module

typescript除了支持const a = 1; export a, 也支持export const a = {}或export = {},但import时必须用import a = require()引入。
