tools

总结postman（latest version v6.4.2）从入门到进阶的一些使用。

界面分部

顶部操作栏为header bar区域，左侧操作栏为slider bar区域，右侧为builder区域。

命名url

builder区域点击save时，在弹出编辑框的Name栏可以命名这个url，方便辨识。

console控制台

底部左下角第三个图标，点击后打开postman console控制台，可以更直观的看到整个请求相应过程。

setting

以下为beader bar部分setting下的general setting部分。

Trim keys and values in request body: 去除请求中的前后空格。
SSL certificate verification: 关闭证书检查。
Language detection: 设置为json后会强制相应头Content-Type为application/json。
Send no-cache header: 保证请求到最新的相应。

关闭启动postman时的create new 提示

点击header bar左上角New，在create new面板底部，取消勾选Show this window at launch。

request

对参数进行url编码

builder中右击参数行，选择encodeURIComponent。

设置cookie

builder中Send按钮下方的cookies按钮可以添加cookie。

预设header

builder中，Headertab下的Presets可以导入预设定的header，方便保存常用的批量header设置。

Content-type和enctype(form-data/urlencoded/Raw/Binary)

Content-type用来告诉server用何种格式解析请求体body，enctype表名请求体body的格式，两者是对应的。form-data可以用来上传文件；urlencoded类似url上拼接的query参数，会对body进行urlencode；raw表名body是纯文本；binary可以用来提交video、audio、image等流数据。

scripts

builder中的Pre-request script 和 test栏可以动态的设置request以及对response进行断言。前者可以在发送前设置request，后者可以在response响应后对response进行接口检验。

re-request script分为三个级别，collection级别，folder级别和request级别，在collection目录、folder目录或者request的...操作按钮中点击Edit，然后选择re-request script即可编辑js脚本。

collection级别的re-request script会在该collection中的每个request运行前执行，folder级别的re-request script会在该collection中的每个request运行前执行，request级别的re-request script会在每个request运行前执行。

test同理。

re-request script

在re-request script中通过设置如:

test

pm2.test('token return correctly', function() {
    pm.response.to.have.status(200);
})

send请求后，test的运行结果会以控制台的形式输出结果。

编辑re-request script和test时，右侧有常用的功能性代码提示，用来设置变量或者进行断言。示例文档中举例了常用的断言方式。

指定collection运行顺序

通过设置：

postman.setNextRequest("request_name");

来指定当前请求执行以后应该执行的下一个请求。

environments and globals

environment、globals代表两类键值对的变量集合。postman可以在不同级别的的scope中创建变量，然后在不同的地方引用这些变量获取预设定的值。例如作用域为global级别的变量集合，作用域为environment级别的变量集合，越往后变量优先级越高。global级别的全局变量集合只能有一个。值的类型都是String，如果需要存储objects或者array，请用JSON序列化。

设置变量。

  • builder右上角的environment后的设置按钮和查看按钮都可以设置不同environment级别变量集合以及global级别变量集合。
  • collection目录右侧...中的edit中在variables下可以设置collection级别的变量。
  • 编辑re-request script和test时，通过pm.environment.set()/pm.globle.set()设置environment级别和globle级别的变量。

获取变量

在不同的地方可以获取到设置的变量值。

  • 通过在编辑URL / headers / body中的值时作为动态参数拿到最近的域内的key值。
  • 通过在pre-request and test scripts中通过pm.environment.get()/pm.environment.get()/pm.variables.get()获取到某个级别或最近级别的变量。
  • script中使用console.log(key)可以在postman console中打印出对应的值。

collection runs

header bar右上角第三个按钮Runner操作按钮，可以对collection内部的requests进行批量执行，同时可以设定environment变量集合、循环次数等，点击Run以后即可看到批量执行后的结果，输出test中的断言。

requests执行顺序

默认collection中的requests是从上到下依次执行的，后一个request是等前一个request的test执行完毕后再发起请求，且同级collection下的folder中的request先执行，在执行collection根目录下的request。

Building workflows（setNextRequest）

可以通过设置postman.setNextRequest(‘request name’)指定下一个request，这之间的request将会被跳过（忽略），然后会继续执行后续的request。

postman.setNextRequest()只会在当前script的末尾执行。

postman.setNextRequest()是有作用域的，collection下的该方法可以指定collection下任意request，但是folder下只能指定当前folder下的request。
