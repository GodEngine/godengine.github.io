更新pm2

pm2 update

PM2 2.4.0以上支持正则表达式命令(eq: pm2 restart /work[1,2,3]/), 只支持name而非id。

集群模式

开启集群模式(设置exec_mode为‘cluster’,默认为’fork’模式)可以充分利用多核CPU的计算优势，提高应用的应能和可靠性，参看node文档中关于集群模式的解释。pm2中开启集群模式的配置为：

{
    "apps" : [{
    "instances" : "max",
    "exec_mode" : "cluster" 
    }]
}

instances为0/max表示应用到所有CPU
instances为-1表示应用到比总数-1的CPU数量上
instances为其他number则代表应用到的CPU数量

进程配置文件

pm2使得可以通过配置文件来管理你的工作流，比如设置每个服务的参数、环境、日志文件等，以及启动，停止等操作，格式可以为JSON、Javascript、YAML。

以下命令会在当前目录下生成一个基础的配置文件：

pm2 ecosystem

生成的内容如下（同时支持json、yaml格式）：

module.exports = {
  apps : [{
    name        : "worker",
    script      : "./worker.js",
    watch       : true,
    env: {
      "NODE_ENV": "development",
    },
    env_production : {
       "NODE_ENV": "production"
    }
  },{
    name       : "api-app",
    script     : "./api.js",
    instances  : 4,
    exec_mode  : "cluster"
  }]
}

此时便可以通过执行该配置文件管理各个服务，如：

pm2 start ecosystem.config.js   #启动所有服务
pm2 start ecosystem.config.js --only app    #只启动app服务
pm2 stop ecosystem.config.js    #停止所有服务

常用的配置参数如下
|字段 |类型 |示例 |含义 |
|:——:|——|——|——–|
|name|string|’webapp’|应用名称|
|script|string|’./path/index.js’|pm2 start启动入口文件|
|instances|number|0/max/-1等|实例的运行数目|
|exec_mode|string|’cluster’|启动模式,’cluster’/‘fork’|
|watch|boolean/[]|true|true则开启监听到文件变化时重启服务|
|ignore_watch|list|[‘node_modules’]|忽略对某些文件或文件夹的监听|
|max_memory_restart|string|’50M’|最大内存值，超过则重启服务|
|env|object|{‘NODE_ENV’: ‘production’, ‘ID’: ‘89’}|设置服务环境|
|env_< environment >|string|’env_production’|命令行中带有–env < environment >会将当前服务的环境变量设置为使用该属性对应的值|
|log_date_format|string|’YYYY-MM-DD HH:mm Z’|日志格式化|
|error_file|string|’/data/log/webapp/error.log’|error日志存放地址|
|out_file|string|’/data/log/webapp/error.log’|out日志存放地址|
|merge_logs|boolean|true|设为true会在日志文件中隐藏项目名后缀|
|source_map_support|boolean|true|是否支持source_map|

优雅的启动/退出

安全退出

如需在关闭进程前做些保存之类的安全操作，可以通过监听SIGINT事件执行相应的处理操作:

process.on('SIGINT', function() {
    db.stop(function(err) {
        ...
    });
});

安全启动

如需在启动进程前做些如确保连接已建立等操作，可以通过监听process.send(‘ready’)事件，且配置文件中已配置‘wait_ready’为true来实现，如下:

var http = require('http');
var app = http.createServer(function(req, res) {
res.writeHead(200);
res.end('hey');
})
var listener = app.listen(0, function() {
console.log('Listening on port ' +             listener.address().port);
  // Here we send the ready signal to PM2
process.send('ready');
});

日志管理

基本的命令

pm2 logs -h   // 显示pm2 logs的可选参数
pm2 logs app    // 显示app服务的15行error和out日志
pm2 logs /app[12]/  // 正则匹配服务名称

PM2 2.x版本以上可以在进程配置文件中通过配置log_type字段实现日志的json格式输出：

"log_type": "json"

清空所有日志：

pm2 flush

推荐使用pm2-logrotate来进行日志管理，该工具提供了设置日志轮转(即设置日志周期)、设置单个日志文件大小、保留日志数目、格式化日志文件名等功能，官方文档有详细的使用方式。

可以在进程文件中设置合并error日志和out日志，如下：

如下会将out日志和error日志都打入combined文件中：
{
    "out_file": "combined.log",
    "error_file": "combined.log"
}

如下会生成out日志和error日志的同事，合并两者到combined文件：
{
    "log_file": "combined.outerr.log",
    "out_file": "out.log",
    "error_file": "err.log"
}

可以通过进程文件配置关闭日志功能：

{
    "out_file": "/dev/null",
    "error_file": "/dev/null"
}

pm2开机自启动

执行pm2 startup script后,按照提示copy/past命令，会根据系统生成自启动的脚本，当所有应用启动后，执行pm2 save，会保存当前的服务列表至$PM2_HOME/.pm2/dump.pm2，下次机器重启是，这些保存的服务服务列表会自启动。需要手动重启这些进程时，即执行pm2 resurrect。

注意，当升级node版本尤其是通过vim升级时，务必更新启动脚本，即执行pm2 unstartup，pm2 startup

停用开机自启动：pm2 unstartup
